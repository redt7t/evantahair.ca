Sea Raiders

Iterative Process

Our iterative process for this project was; build a low-fidelity prototype, design and build a high-fidelity prototype, then conduct user-testing and create a final version that incorporates any feedback.

User Testing

We conducted user testing to determine what users thought about our game and what we could improve on. The quantitative feedback indicated that the difficulty of the game was well balanced, so we did not need to adjust it for the final iteration. The qualitative feedback indicated that users did not understand what the game controls were, so we added the controls to the title screen.

My Roles

I was the lead programmer on this project and played a supporting role as a game artist, while my partner was the inverse.

Play the Game

Sea Raiders is a game where you play as a pirate ship and you must defend your seas from incoming enemy ships. If you allow too many to get behind you, you will lose. Survive the onslaught long enough and you will face the enemy's largest ship, which you must defeat to win the game. The game can be played on desktop or mobile.
